El reto

Cuando el producto es artesanal, la fotografía pesa. Cada pieza necesita verse con detalle, en alta resolución, con la veta de la madera definida. El problema obvio: las imágenes pesadas matan la performance, y los sitios lentos matan la conversión. En estudios de diseño esto se nota más: quien busca mobiliario único entra desde el móvil, espera 3 segundos como mucho, y se va.

El reto técnico era claro. Mostrar la pieza con peso visual real, que el sitio cargara rápido en una conexión 4G de barrio, y que Google lo recompensara con buen posicionamiento orgánico.

Cómo lo abordamos

Mobile-first de verdad, no como etiqueta de venta. Diseñamos toda la experiencia desde el viewport del móvil hacia arriba. Esto cambia decisiones desde el inicio: peso del bundle, jerarquía visual, qué información se ve en el primer scroll, qué se posterga.

Cada imagen pasa por un pipeline de optimización antes de subirse. Formatos modernos (WebP y AVIF con fallback a JPG), tamaños responsivos servidos con srcset, lazy loading nativo para todo lo que no esté above the fold. El navegador descarga sólo lo que necesita, en la resolución exacta para la pantalla del usuario.

Qué se hizo en concreto

  • Pipeline de imágenes optimizado. Compresión WebP con calidad visual indistinguible, dimensiones servidas según viewport, lazy loading nativo. Las imágenes que antes pesaban 2 MB pasaron a pesar 180 KB sin pérdida perceptible.
  • CSS crítico inline. El CSS necesario para renderizar la primera vista va inline en el HTML. El navegador no espera al CSS externo para pintar. Resultado: First Contentful Paint en menos de 800ms.
  • Fonts con display:swap. La tipografía custom no bloquea el render. El texto aparece con fallback del sistema y se sustituye cuando carga la fuente real, sin Cumulative Layout Shift.
  • JavaScript no bloqueante. Todo el JS no esencial se carga con defer. Las animaciones se ejecutan después de que el contenido principal esté visible, no antes.
  • Cache headers de Cloudflare configurados. Recursos estáticos cacheados al máximo, HTML con cache corto. La segunda visita es prácticamente instantánea.

Resultados técnicos

  • PageSpeed Insights: 98/100 en mobile, 100/100 en desktop.
  • Largest Contentful Paint (LCP): 1.2s en móvil.
  • Cumulative Layout Shift (CLS): 0.0 (perfecto).
  • Interaction to Next Paint (INP): menos de 100ms.
  • Accesibilidad WCAG 2.2 AA: validada.

La performance se evalúa de forma directa: Lighthouse devuelve el mismo número en cada test, repetible y verificable. Los números de negocio del lado del cliente (consultas que llegan, peticiones de presupuesto) son visibles desde el GA4 que dejamos configurado y evolucionan según el tráfico que reciba el sitio.
